## Why

年轻情侣需要一个**重情感陪伴、轻工具属性**的私人空间来记录和经营恋爱关系。现有工具要么偏工具化（日历、备忘录），要么缺乏双人联动设计。我们打造一款以 Hello Kitty 粉色主题为视觉核心、以桌宠陪伴为情感纽带的双人互动 APP，融合日常记录、趣味互动、AI 陪伴三大核心能力，让每对情侣拥有一个专属的「恋爱小宇宙」。

## What Changes

- 新增用户注册登录系统（手机号/邮箱/微信三种方式），配套恋爱码绑定机制
- 新增恋爱码系统，6 位全局唯一码（排除易混淆字符），支持二维码扫码绑定、手动刷新
- 新增首页仪表盘，展示每日情话、打卡进度、待办概览
- 新增每日情话模块，双方可编辑当日文案，WebSocket 实时同步
- 新增情侣状态系统，四大类预设状态（心情/活动/天气/特殊），支持自定义，24h 自动过期，桌宠动作联动
- 新增心情温度计，0-100 滑动条 + 温度计可视化（水银柱+圆球+刻度），五段颜色渐变，情侣心情对比
- 新增天气模块，基于 GPS 定位展示实时天气、三日预报和贴心提醒
- 新增经期管理模块，支持经量/症状/情绪记录，取最近 6 次周期平均预测下次日期
- 新增 AI 恋爱管家模块，基于 5 项输入数据生成今日建议、避坑提醒和话术模板，24h 缓存
- 新增任务中心，支持双人共享习惯打卡（默认 5 项）和待办事项
- 新增 4 款情侣联机小游戏，含默契度评级（灵魂伴侣/心有灵犀/渐入佳境/还需磨合）
- 新增共同桌宠「暹暹」（暹罗猫），四维属性系统 + 每小时衰减 + 等级经验 + 情侣状态联动
- 新增纪念日系统，支持多纪念日、每年重复、多级提醒、手机日历同步
- 新增心愿小纸条，支持悄悄话即时推送和心愿清单
- 新增共享相册，支持多相册分类、点赞（防重复）、楼中楼评论
- 新增两人地图，支持实时位置共享和距离显示

## Capabilities

### New Capabilities

- `auth-system`: 用户注册登录（手机/邮箱/微信）、情侣配对绑定、个人资料管理
- `love-code-system`: 6 位恋爱码生成算法，排除易混淆字符，全局唯一校验，二维码展示，每月限刷新 3 次
- `dashboard`: 首页仪表盘框架，恋爱天数计数器、每日情话、快捷入口、打卡进度、待办概览、天气/经期预览
- `daily-greeting`: 每日情话模块，双方可编辑文案、自定义背景图、WebSocket 实时同步
- `couple-status`: 情侣状态系统，四大类预设 + 自定义状态，24h 自动过期，桌宠动作联动，戳一戳/抱抱/留言互动
- `mood-thermometer`: 心情温度计，0-100 滑动条调节，温度计可视化（水银柱+圆球+刻度），五段颜色渐变，历史趋势图表，情侣心情对比
- `weather-system`: GPS 定位天气、实时温度展示、体感/湿度/风力、三日预报、AI 贴心提醒
- `period-tracker`: 经期记录（经量/症状/情绪/备注）、周期预测（最近 6 次平均 + 置信度）、护理指南
- `ai-companion`: AI 恋爱管家，5 项输入 → 3 条建议 + 1 条避坑 + 2 条话术，Redis 24h 缓存
- `task-center`: 双人习惯打卡（默认 5 项）、共享待办事项、环形进度条统计
- `couple-games`: 4 款联机小游戏 + 默契度评级，房间制 + WebSocket 实时通信
- `shared-pet`: 共同桌宠「暹暹」，四维属性 + 每小时衰减公式 + 四种互动 + 等级经验系统 + 情侣状态联动
- `anniversaries`: 纪念日 CRUD、每年重复/一次性、多级提醒（当天/前 3 天/前 7 天）、手机日历同步
- `wish-notes`: 心愿小纸条（悄悄话即时推送 + 心愿清单管理 + 已读状态）
- `shared-album`: 双人共享相册，多相册分类、批量上传、点赞（防重复）、楼中楼评论
- `couple-map`: 双方实时位置共享、距离显示、位置隐私控制

## Impact

- 技术栈：Flutter 3.x（Riverpod + GoRouter）+ Node.js/NestJS 后端 + MySQL 8.0 + Redis 7.x + Socket.io
- 第三方服务：和风天气 API、高德地图 SDK、极光推送、阿里云 OSS、AI 大模型 API
- 目标平台：iOS + Android
- 数据库：13+ 张核心表，所有业务表依 coupleId 隔离
- 开发周期：6 阶段约 15 周，4 人团队
